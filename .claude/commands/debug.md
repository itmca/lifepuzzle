에러나 버그를 분석하고 해결합니다.
디버깅은 인터랙티브하게 진행됩니다.

## 워크플로우

### 1. 정보 수집

사용자에게 필요한 정보를 요청합니다:
- 에러 메시지, 스택 트레이스
- 재현 단계
- 예상 동작 vs 실제 동작

### 2. 로그 추가 제안

원인 파악을 위해 로그 추가가 필요하면:
- 추가할 로그 위치와 내용을 제안
- **사용자 확인 후** 로그 추가
- 사용자가 실행하고 결과 공유

```typescript
// RN 예시
console.log('[DEBUG] 변수명:', 변수);
logger.debug('[DEBUG] 상태:', state);
```

```java
// Spring 예시
log.debug("[DEBUG] 파라미터: {}", param);
```

### 3. 분석 및 해결

로그 결과를 바탕으로:
- 원인 분석
- 수정 방안 제안
- **사용자 확인 후** 코드 수정

### 4. 디버그 로그 제거

문제 해결 후:
- 추가했던 디버그 로그 모두 제거
- 정리된 코드 확인

```bash
# 추가한 DEBUG 로그 검색
git diff | grep -E "\[DEBUG\]"
```
